# قواعد العمل في هذا المستودع

## سير عمل Git / PR — إلزامي

- **قبل أي `git push` لاحق إلى فرع له PR:** تحقّق أولاً من حالة الـ PR (`merged`/`closed`/`open`).
  - إذا كان الـ PR **مدموجاً أو مغلقاً**: لا تدفع المزيد إلى نفس الفرع ولا تقل "دُفعت إلى PR رقم كذا".
    افتح **PR جديداً** للعمل الجديد فوق `main`.
  - إذا كان الـ PR **مفتوحاً**: ادفع إليه عادياً.
- لا تطلب من المستخدم الموافقة على PR سبق دمجه.

## ملاحظات المشروع

- لعبة ويب (Three.js) في ملف واحد `index.html`، تعمل كـ PWA دون اتصال.
- لا أصول خارجية (لا GLB ولا صور خامات) — كل الرسومات إجرائية / مولّدة بالكانفس وقت التشغيل.
- بيئة التطوير بلا متصفح ولا شبكة → لا يمكن التحقق البصري؛ تحقّق من بناء جملة JS عبر `node --check`،
  وأحِط الميزات المتقدمة بحراسات `try/catch` ترجع لمسار بسيط عند الفشل.
